# Sample Requirements: Subscription Billing Manager

Build a subscription billing manager for a SaaS product.

Each subscription should include subscription ID, customer name, email, plan name, billing cycle, monthly amount, start date, renewal date, status, and notes.

The system should allow users to add subscriptions, cancel subscriptions, reactivate subscriptions, update plan, list all subscriptions, filter by status, filter by plan, calculate monthly recurring revenue, calculate active subscriptions, identify upcoming renewals, and generate a billing summary.

Validate email, positive monthly amount, supported plan/status/billing cycle, valid renewal date, and non-existing subscription operations.

The Gradio app should display subscriptions and upcoming renewals as tables. MRR and billing summary should be shown as readable Markdown metrics and tables.
