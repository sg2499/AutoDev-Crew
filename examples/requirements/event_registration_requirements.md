# Sample Requirements: Event Registration Manager

Build an event registration management app.

Each registration should include registration ID, attendee name, email, phone, event name, ticket type, payment status, registration date, check-in status, and notes.

The system should create registrations, update payment status, mark check-in, cancel registration, list registrations, filter by event, filter by ticket type, filter by check-in status, calculate total attendees, calculate revenue, and generate event summary.

Validate email, phone, supported ticket types/statuses, non-empty names, duplicate IDs, and non-existing registrations.

The Gradio app should show registrations, filtered lists, and check-in lists as tables. Event summaries should be Markdown metrics and tables.
