# Sample Requirements: Restaurant Order Manager

Build a restaurant order management app.

Each order should include order ID, table number, customer name, order items, quantity, item price, total amount, order status, payment status, order time, and notes.

The system should create orders, update order status, update payment status, cancel orders, list all orders, filter by status, filter by payment status, calculate total sales, identify unpaid orders, and generate daily order summary.

Prevent invalid table numbers, empty item lists, negative prices/quantities, unsupported statuses, and updates to non-existing orders.

The Gradio app should show orders, filtered orders, and unpaid orders as tables. Daily sales summary should be shown as readable metrics and Markdown.
