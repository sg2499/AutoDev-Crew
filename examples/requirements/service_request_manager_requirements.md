# Sample Requirements: Home Service Request Manager

Build a home service request management app.

Each service request should include request ID, customer name, phone, service type, issue description, preferred date, preferred time, assigned technician, status, service charge, and notes.

The system should create requests, assign technician, update status, list requests, filter by service type, filter by technician, filter by status, identify pending requests, calculate expected revenue, and generate service operations summary.

Prevent invalid phone numbers, empty customer names/descriptions, negative charges, unsupported statuses, and updates to non-existing requests.

The Gradio app should show service requests, pending requests, and filtered results as tables. Operations summaries should be Markdown reports.
