# Sample Requirements: Library Book Lending Manager

Build a library book lending management app.

Each book should include book ID, title, author, category, ISBN, total copies, available copies, and shelf location. Each loan should include loan ID, book ID, borrower name, borrower contact, issue date, due date, return date, and status.

The system should allow users to add books, update copies, issue books, return books, list books, list active loans, filter books by category, search books by title/author, identify overdue loans, and generate library usage summary.

Prevent issuing unavailable books, invalid copy counts, empty titles, invalid due dates, returning non-active loans, and duplicate IDs.

The Gradio app should show books, loans, overdue loans, and search results as tables. Usage summaries should be readable reports.
