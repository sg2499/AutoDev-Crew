# Sample Requirements: Clinic Appointment Scheduler

Build an appointment scheduling app for a small clinic.

Each appointment should include appointment ID, patient name, phone number, doctor name, department, appointment date, appointment time, status, and notes.

The system should allow users to create, reschedule, cancel, confirm, and complete appointments. It should list all appointments, filter by doctor, filter by department, filter by date, show upcoming appointments, and generate a daily schedule report.

The system should prevent duplicate appointments for the same doctor at the same date and time, invalid dates, empty patient names, unsupported statuses, and updates to non-existing appointments.

The Gradio app should show appointment lists and schedules as tables. Daily schedule reports and status summaries should be readable Markdown or summary tables.
