# Sample Requirements: Course Enrollment Manager

Build a course enrollment management app for a training institute.

The system should allow users to add courses, enroll students, update payment status, update enrollment status, list enrollments, filter by course, filter by payment status, calculate total enrollments, calculate revenue collected, and generate a course-wise enrollment summary.

Each enrollment should include enrollment ID, student name, email, phone, course name, batch name, enrollment date, fee amount, payment status, enrollment status, and notes.

The system should validate email, phone, non-negative fee amount, supported payment statuses, supported enrollment statuses, and prevent duplicate enrollment IDs.

The Gradio app should display enrollment records and filtered records as tables. Revenue and course summaries should be readable Markdown and summary tables.
