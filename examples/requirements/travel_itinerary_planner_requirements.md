# Sample Requirements: Travel Itinerary Planner

Build a travel itinerary planning app.

Each itinerary item should include item ID, destination, date, time, activity name, activity type, estimated cost, booking status, location/address, and notes.

The system should add itinerary items, update booking status, delete items, list itinerary, filter by destination, filter by date, calculate total estimated cost, identify unbooked activities, and generate day-wise travel summary.

Prevent invalid dates/times, negative costs, empty destinations/activities, unsupported booking statuses, and updates to non-existing items.

The Gradio app should show itinerary items and unbooked activities as tables. Day-wise summaries should be readable Markdown.
