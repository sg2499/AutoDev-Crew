# Sample Requirements: Meeting Action Item Tracker

Build a meeting action item tracking app.

Each action item should include action ID, meeting title, action description, owner, assigned date, due date, priority, status, completion date, and notes.

The system should create action items, update status, assign owner, delete actions, list all actions, filter by owner, filter by status, identify overdue actions, calculate completion rate, and generate meeting follow-up summary.

Prevent due dates before assigned dates, empty action descriptions, unsupported priorities/statuses, duplicate IDs, and updates to non-existing actions.

The Gradio app should show action items and overdue actions as tables. Follow-up summaries should be readable Markdown.
