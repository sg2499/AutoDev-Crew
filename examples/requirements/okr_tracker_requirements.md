# Sample Requirements: OKR Tracker

Build an OKR tracking app for teams.

Each objective should include objective ID, objective title, owner, quarter, status, progress percentage, and notes. Each key result should include key result ID, objective ID, description, target value, current value, unit, status, and confidence score.

The system should create objectives, add key results, update progress, update status, list OKRs, filter by owner, filter by quarter, identify at-risk OKRs, calculate average progress, and generate OKR health summary.

Prevent invalid progress percentage, invalid confidence score, empty objective titles, unsupported statuses, and key results for non-existing objectives.

The Gradio app should display OKRs, key results, and at-risk OKRs as tables. Health summaries should be readable Markdown.
