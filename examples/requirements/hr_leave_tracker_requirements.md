# Sample Requirements: HR Leave Request Tracker

Build an HR leave request tracking app.

Each leave request should include request ID, employee name, employee ID, department, leave type, start date, end date, number of days, reason, status, approver, and comments.

The system should allow users to submit leave requests, approve/reject requests, update status, list all requests, filter by employee, filter by department, filter by status, calculate total leave days, identify pending approvals, and generate a department-wise leave summary.

Prevent invalid date ranges, empty employee names, unsupported leave types/statuses, negative leave days, duplicate request IDs, and updates to non-existing requests.

The Gradio app should show leave requests, pending approvals, and filtered results as tables. Summary outputs should be readable reports.
