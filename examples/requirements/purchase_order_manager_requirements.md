# Sample Requirements: Purchase Order Manager

Build a purchase order management app.

Each purchase order should include PO ID, supplier name, item name, quantity, unit price, order date, expected delivery date, approval status, delivery status, and notes.

The system should create purchase orders, update approval status, update delivery status, delete orders, list all orders, filter by supplier, filter by approval status, identify pending approvals, calculate total order value, and generate supplier-wise purchase summary.

Prevent negative quantity/price, invalid dates, unsupported statuses, empty supplier/item names, and non-existing order updates.

The Gradio app should show purchase orders, filtered orders, and pending approvals as tables. Purchase summaries should be readable Markdown.
