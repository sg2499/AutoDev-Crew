# Sample Requirements: Warehouse Dispatch Tracker

Build a warehouse dispatch tracking app.

Each dispatch should include dispatch ID, order ID, customer name, item name, quantity, dispatch date, carrier, tracking number, delivery status, expected delivery date, and notes.

The system should create dispatch records, update delivery status, list dispatches, filter by carrier, filter by delivery status, identify delayed deliveries, calculate dispatched quantities, and generate dispatch summary.

Prevent negative quantities, invalid dates, empty customer/item names, unsupported delivery statuses, duplicate IDs, and non-existing updates.

The Gradio app should show dispatch records and delayed deliveries as tables. Dispatch summaries should be Markdown reports.
