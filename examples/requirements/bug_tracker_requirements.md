# Sample Requirements: Bug Tracking System

Build a bug tracking app for a software team.

Each bug should include bug ID, title, description, severity, priority, status, assigned developer, reported by, created date, due date, and resolution notes.

The system should create bugs, update status, update priority, assign developer, close bugs, list all bugs, filter by severity, filter by status, filter by developer, identify overdue bugs, and generate defect summary.

Prevent empty titles/descriptions, unsupported severity/priority/status, due date before created date, and updates to non-existing bugs.

The Gradio app should display bugs, filtered bugs, and overdue bugs as tables. Defect summaries should be readable Markdown.
