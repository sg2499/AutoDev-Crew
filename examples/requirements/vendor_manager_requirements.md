# Sample Requirements: Vendor Performance Manager

Build a vendor performance management app.

Each vendor should include vendor ID, vendor name, category, contact email, phone, rating, total orders, on-time deliveries, delayed deliveries, active status, and notes.

The system should add vendors, update ratings, update delivery stats, deactivate vendors, list vendors, filter by category, filter by active status, calculate on-time delivery rate, identify low-performing vendors, and generate vendor performance summary.

Validate email, phone, rating range, non-negative delivery counts, duplicate IDs, and non-existing vendors.

The Gradio app should show vendor records and low-performing vendors as tables. Performance summaries should be Markdown and summary tables.
