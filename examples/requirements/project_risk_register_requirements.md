# Sample Requirements: Project Risk Register

Build a project risk register app.

Each risk should include risk ID, project name, risk title, risk description, probability, impact, risk score, owner, mitigation plan, status, review date, and notes.

The system should add risks, update probability/impact, update status, assign owner, list all risks, filter by project, filter by status, identify high-risk items, calculate average risk score, and generate project risk summary.

Validate probability and impact ranges, non-empty fields, supported statuses, valid review dates, duplicate IDs, and non-existing risks.

The Gradio app should show risk records, high-risk records, and filters as tables. Risk summaries should be Markdown reports.
