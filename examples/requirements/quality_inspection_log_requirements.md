# Sample Requirements: Quality Inspection Log

Build a quality inspection logging app.

Each inspection should include inspection ID, product name, batch number, inspector name, inspection date, defect count, sample size, pass/fail status, defect category, corrective action, and notes.

The system should add inspections, update corrective action, update status, list inspections, filter by product, filter by pass/fail status, identify high-defect batches, calculate defect rate, and generate quality summary.

Prevent negative defect counts, defect count greater than sample size, invalid sample size, unsupported statuses, and updates to non-existing inspections.

The Gradio app should display inspections and high-defect batches as tables. Quality summaries should be Markdown reports.
