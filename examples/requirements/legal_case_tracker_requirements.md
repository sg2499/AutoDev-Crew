# Sample Requirements: Legal Case Tracker

Build a legal case tracking app for a small office.

Each case should include case ID, client name, case title, case type, filing date, next hearing date, court name, assigned lawyer, status, and notes.

The system should add cases, update hearing date, update status, assign lawyer, list cases, filter by lawyer, filter by case type, filter by status, identify upcoming hearings, and generate case workload summary.

Prevent invalid dates, empty client/case names, unsupported statuses, duplicate IDs, and updates to non-existing cases.

The Gradio app should display cases and upcoming hearings as tables. Workload summaries should be readable Markdown.
