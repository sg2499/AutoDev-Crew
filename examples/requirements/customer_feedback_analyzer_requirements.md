# Sample Requirements: Customer Feedback Manager

Build a customer feedback management app.

Each feedback record should include feedback ID, customer name, email, product/service name, rating, feedback text, category, sentiment label, received date, status, and notes.

The system should add feedback, update status, assign sentiment manually, list feedback, filter by rating, filter by sentiment, filter by category, identify low-rating feedback, calculate average rating, and generate feedback summary.

Prevent invalid email, rating outside 1-5, empty feedback text, unsupported statuses/sentiments, and updates to non-existing feedback.

The Gradio app should display feedback records, low-rating feedback, and filtered results as tables. Summary outputs should be readable Markdown metrics.
