# Sample Requirements: Parcel Tracking Manager

Build a parcel tracking management app.

Each parcel should include tracking ID, sender name, recipient name, origin city, destination city, parcel weight, shipping date, expected delivery date, current status, carrier, and notes.

The system should add parcels, update status, list parcels, filter by carrier, filter by status, identify delayed parcels, calculate total parcels by status, and generate logistics summary.

Prevent negative weight, invalid date ranges, empty sender/recipient names, unsupported statuses, duplicate tracking IDs, and updates to non-existing parcels.

The Gradio app should display parcels and delayed parcels as tables. Logistics summaries should be Markdown reports.
