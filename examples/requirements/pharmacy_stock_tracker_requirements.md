# Sample Requirements: Pharmacy Stock Tracker

Build a pharmacy stock tracking app.

Each medicine record should include medicine ID, medicine name, batch number, category, quantity, reorder level, expiry date, supplier, unit price, and notes.

The system should add medicines, update stock, delete medicines, list medicines, filter by category, identify low-stock medicines, identify near-expiry medicines, calculate inventory value, and generate reorder summary.

Prevent negative quantities/prices, invalid expiry dates, empty medicine names, duplicate batch IDs, and updates to non-existing medicines.

The Gradio app should show medicine records, low-stock items, and near-expiry items as tables. Inventory and reorder summaries should be Markdown reports.
