# Sample Requirements: Student Behavior Observation Log

Build a student behavior observation logging app for teachers.

Each observation should include observation ID, student name, class, date, behavior category, observation notes, teacher action, follow-up required, follow-up date, and status.

The system should add observations, update status, mark follow-up complete, list observations, filter by student, filter by category, filter by follow-up status, identify pending follow-ups, and generate student observation summary.

Prevent empty student names, invalid dates, unsupported categories/statuses, follow-up date before observation date, and updates to non-existing observations.

The Gradio app should show observations and pending follow-ups as tables. Student summaries should be readable Markdown reports.
